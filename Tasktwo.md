## №4

**Приоритет:** High

**Название:** Граничные значения параметра *q* при работе АПИ для выдачи регионов методом GET 

**Предусловия:** В раздел Params внесен параметр *key - q.* Параметр *q* должен содержать не менее 3 символов. Возьмем регион с самым длинным названием *Петропавловск-Камчатский*. <br>  При корректном ответе тело будет выглядеть следующим образом:
<pre><code>
{
    "total": 22,
    "items": [
        {
            "id": 95,
            "name": "Петропавловск-Камчатский",
            "code": "p_kamchatskiy",
            "country": {
                "name": "Россия",
                "code": "ru"
            }
        }
    ]
}
</code></pre>
В ином случае пишем какую ошибку нам выдало. 

**Тестовые данные:** <br> 1. *key - q | value - кий* <br> 2. *key - q | value - ский* <br> 3. *key - q | value - ий* <br> 4. *key - q | value - Камчатский* <br> 5. *key - q | value - етропавловск-Камчатский* <br> 6. *key - q | value - Петропавловск-Камчатский* <br> 7. *key - q | value - Ппетропавловск-Камчатский*

**Шаги:** 
1. Выбор метода GET, строка url - https://regions-test.2gis.com/1.0/regions
2. Раздел Params, ввод тестовых данных 1, отправка запроса
3. Раздел Params, ввод тестовых данных 2, отправка запроса
4. Раздел Params, ввод тестовых данных 3, отправка запроса
5. Раздел Params, ввод тестовых данных 4, отправка запроса
6.  Раздел Params, ввод тестовых данных 5, отправка запроса
7.  Раздел Params, ввод тестовых данных 6, отправка запроса
8.  Раздел Params, ввод тестовых данных 7, отправка запроса


**Ожидаемый результат:**
